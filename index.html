What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Delivery or Deployment. It is a modern way to build, test, and release software faster and more reliably. Continuous Integration means developers frequently push code changes to a shared repository. Continuous Delivery means the application is automatically prepared for release after testing. This helps reduce errors, improve quality, and make updates easier and faster.

My Pipeline

After I push my code to GitHub, the repository stores the latest version of my project. Then the workflow can run automatically to check the code or deploy the website. If everything works correctly, the updated site becomes available online.
